What is an Exception?

An exception is an object that signals an error during program execution. When an exception is raised, by default, Python stops executing the current code and displays an error message. This error message contains valuable information about what went wrong and where it happened.

Reading Error Messages

Let's look at an example. If you try to use a variable that hasn't been defined:

print(a) print("This line never runs")

You'll see an error message like:

Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
    print(a)
NameError: name 'a' is not defined

Breaking down this error message:

  • Traceback: Shows the sequence of function calls leading to the error
  • File and line number: Points to exactly where the error occurred
  • The problematic line: Shows the code that caused the error
  • Exception type: The name of the exception (in this case, NameError)
  • Error description: A message explaining what went wrong

Common Exception Types

SyntaxError

A SyntaxError occurs when Python code violates the language's syntax rules. Python detects these before running your code:

x = 5 y = x +

NameError

A NameError occurs when you try to use a variable that hasn't been defined:

print(undefined_variable)

TypeError

A TypeError occurs when you try to perform an operation on an object of the wrong type:

print("5" + 3)

Always Read Exception Messages

When you encounter an exception, read the error message carefully. It usually tells you:

  • What type of error occurred
  • Where it occurred (file and line number)
  • Why it occurred (the description)

Understanding error messages is one of the most important skills in programming. Most errors are fixable once you understand what went wrong.
